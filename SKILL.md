# stamp-spec — LINE スタンプ設計書 量産システム

このスキルは、LINE スタンプ（LINE Creators Market 審査対象）の設計書を、AskUserQuestion による対話ヒアリングだけで一つずつ確定させながら `~/stamp-specs/<スタンプ名>.md` に**1個のMarkdownファイル**として出力する。

最終成果物（設計書）は、画像生成AI（**ChatGPT (GPT-4o image / DALL-E)** および **Stable Diffusion / NovelAI**）に**1回入力するだけで審査通過品質の画像が1発で出力される**ことを目標とする。曖昧さを残さない。

---

## 起動時の原則

1. **必ず AskUserQuestion で完結する。** 憶測で埋めない。「推測で埋めて」と明示された項目のみ推測してよい。
2. **フェーズ固定・順序固定。** 下記の6フェーズを上から順に実行する。各フェーズ内は最大4問の AskUserQuestion を1回呼ぶ（必要なら同フェーズ内で再度呼ぶ）。
3. **LINE 審査ガイドライン完全網羅。** サイズ・余白・著作権・禁止表現を全項目チェックリスト化して設計書に明記する。1項目たりとも省略しない。
4. **画像生成AIは2系統並列。** 1つの設計書に **ChatGPT/DALL-E 用の自然言語プロンプト** と **Stable Diffusion / NovelAI 用のタグ列プロンプト** の両方を必ず併記する。
5. **デフォルト40枚。** ユーザーが他枚数を希望しない限り、フルセット40枚分のプロンプトを生成する。
6. **キャラ確定方式は両対応。** 画像が提供されたら Read してビジュアル特徴を抽出し設計書化する。提供なしならフェーズ1で AskUserQuestion で詳細ヒアリングする。

---

## フェーズ0：スタンプ名・キャラ画像の有無確定（最優先）

最初の発話：「LINE スタンプ設計書を対話で作ります。スタンプ名と、キャラクター画像があるかどうかから確認します」とだけ述べ、即 AskUserQuestion を呼ぶ。

聞く内容：

1. **スタンプ名（Other で自由入力前提）**：日本語名・英文ローマ字名どちらでも可。これがファイル名に使われる。
2. **キャラクター画像の有無**：「これからアップロード/パスを伝える」／「画像なし、ゼロから設計する」／「テキストスタンプで キャラなし」
3. **総枚数**：8 / 16 / 24 / 32 / **40（推奨・デフォルト）**
4. **タッチ（画風）の方向性**：ゆるかわ手描き / シンプルベクター / リアル系 / アニメ調 / ピクセルアート / 写真風 / その他

スタンプ名確定後、即座に：
```bash
mkdir -p ~/stamp-specs
```

以降、フェーズ1〜6が完了するたびに **~/stamp-specs/<スタンプ名>.md を逐次追記・更新** する（最後に一括ではなく部分保存）。

**画像が提供されると言われた場合**：ユーザーから画像パスを受け取り Read で読み込む。読み込んだ内容（色・体型・特徴的パーツ・線質・塗り方）をフェーズ1で設計書化する。

## フェーズ1：キャラクター設定（画像なしの場合は全項目ヒアリング、画像ありの場合は補完項目のみ）

### 画像なしルート

AskUserQuestion を最大2回（合計最大8問）呼ぶ。

**1回目（4問）**：

1. **種族・モチーフ**：人間／犬／猫／うさぎ／鳥／架空生物／無生物（食べ物・物体）／その他
2. **体型プロポーション**：2〜3頭身（デフォルメ強）／4〜5頭身（中間）／6頭身以上（リアル寄り）／不定形
3. **ベースカラー**：白／黒／茶／パステル系／ビビッド系／モノクロ／カラフル混在
4. **性格イメージ**：ゆる／元気／クール／毒舌／天然／真面目／その他（スタンプの台詞トーンに直結）

**2回目（4問）**：

5. **顔の特徴（multiSelect）**：丸目／点目／ジト目／キラキラ目／半目／涙目／眉なし／頬染め固定／口大きめ／口小さめ／牙／ヒゲ
6. **特徴的パーツ（multiSelect）**：帽子／リボン／メガネ／首輪／しっぽ／羽／角／模様（ぶち・しま）／なし
7. **線画スタイル**：太線でクッキリ（推奨：LINE標準）／細線で繊細／線なし（フラット）／鉛筆風ラフ
8. **塗りスタイル**：ベタ塗り（推奨：LINE標準）／グラデーション／水彩風／陰影あり（セルシェード）／テクスチャあり

### 画像ありルート

提供画像を Read した結果から自動抽出した特徴を提示しつつ、不明点のみ AskUserQuestion で確認する。最低限聞く項目：

1. **抽出特徴の確認**：「画像から○○・△△・□□を抽出しました。これで設計書に確定してよいか？」（Yes/補正/別画像参照）
2. **画像で表現されていない要素の補完**：性格／台詞トーン／追加ポーズの自由度

## フェーズ2：スタンプ40枚（または指定枚数）の構成決定

### ラウンド2-A：用途カテゴリ配分

AskUserQuestion で以下を聞く（4問）。

1. **配分方針**：定番網羅型（挨拶・返事・感情・行動を均等）／感情特化型（喜怒哀楽中心）／ビジネス特化／推し活・ファン向け／ネタ・ギャグ特化／ニッチ特化（ユーザー指定）
2. **必須カテゴリ（multiSelect）**：挨拶（おはよう・おやすみ等）／返事（はい・いいえ・OK・了解）／感情（喜・怒・哀・楽・驚）／謝罪・お礼／応援／日常行動（食べる・寝る・走る）／季節（誕生日・正月・クリスマス）／ビジネス（了解・承知・お疲れ様）／ネタ／無言（リアクション）
3. **テキスト有無の比率**：全枚数テキスト入り／約半分テキスト入り／テキスト最小限（数枚のみ）／テキストなし（イラストのみ）
4. **言語**：日本語のみ／英語のみ／日本語＋英語ミックス／無言主体

### ラウンド2-B：各スタンプの具体構成

ラウンド2-A確定後、設計書に **40枚分（または指定枚数分）の表** を機械的に展開する。各行に：

| # | カテゴリ | 台詞（あれば） | ポーズ・表情 | キーアクション |

ユーザーに台詞・ポーズの**全件自動生成 vs 主要ポイントのみ確認**を AskUserQuestion で聞く。

1. **生成方針**：全40枚自動生成（ユーザーは後で個別修正）／主要10枚はユーザーと一緒に詰める／全件1枚ずつ確認

## フェーズ3：LINE 審査ガイドライン完全網羅チェックリスト

ヒアリング不要。設計書に**固定で全項目を埋め込む**。生成画像が以下を満たすことをプロンプトに反映する。

### 3-1. 規格仕様（LINE Creators Market 公式・必須）

| 項目 | 要件 |
|---|---|
| メイン画像 | 240 × 240 px / PNG / 透明背景 |
| スタンプ画像 | 370 × 320 px（最大） / PNG / 透明背景 |
| トークルームタブ画像 | 96 × 74 px / PNG / 透明背景 |
| 余白 | 画像端から最低 **10 px** の余白を確保（被写体が端に接触しない） |
| 解像度 | 72 dpi 以上 |
| カラーモード | RGB |
| ファイル形式 | PNG（透過情報含む） |
| 背景 | 完全透明（半透明・縁取り背景・色付き背景は不可） |
| 1枚あたり最大ファイルサイズ | 1 MB 以内 |
| 総枚数選択肢 | 8 / 16 / 24 / 32 / 40 のいずれか |

### 3-2. 著作権・肖像権チェック（全件NG）

- [ ] **既存キャラクター**の引用・模倣・パロディ（他社IP・アニメ・漫画・ゲーム）
- [ ] **実在の有名人**の写真・似顔絵・名前
- [ ] **企業ロゴ・ブランドマーク**（Apple ロゴ・Nike ロゴ等）
- [ ] **第三者の写真**（友人・他人を撮影した写真）
- [ ] **既存の商業作品の構図・名シーン**の流用
- [ ] **トレース疑義**を生む既存画像参照

→ 設計書には「**完全オリジナルキャラ・オリジナル構図のみ**」を明記し、画像生成AIにも `original character, no copyrighted material, no real persons, no brand logos` を必ずプロンプト末尾に含める。

### 3-3. 禁止表現チェック（全件NG）

- [ ] 暴力・流血・武器使用の生々しい描写
- [ ] 性的・露骨な描写、過度な露出
- [ ] 差別表現（人種・性別・宗教・職業）
- [ ] 政治的主張・特定政党支持
- [ ] 宗教的シンボルの礼賛・揶揄
- [ ] 違法行為（薬物・喫煙の未成年・飲酒の未成年）
- [ ] 自殺・自傷の助長
- [ ] 他者を不快にさせるスラング・侮辱語
- [ ] 個人情報（電話番号・住所・口座等）
- [ ] QRコード・バーコード（読み取り誘導）
- [ ] 他社サービス名・商品名の言及
- [ ] LINE 公式マーク・LINE ロゴの模倣

→ 設計書末尾の「ネガティブプロンプト」に全項目を反映する（後述フェーズ5）。

### 3-4. テキスト要件

- [ ] 文字スタンプは**読みやすさ優先**（小さすぎ・装飾過多で潰れない）
- [ ] **誤字脱字なし**
- [ ] 日本語スタンプ内に外国語混在は最小限（英単語1〜2語までを目安）
- [ ] スラング・若者言葉は使ってよいが**侮辱語は不可**

## フェーズ4：画風・カラーパレット・統一仕様の確定

AskUserQuestion で以下を聞く（4問）。

1. **カラーパレット指定**：3色まで指定（Other 自由入力で「#hex」形式可）／系統指定（パステル／ビビッド／モノトーン）／キャラ画像に準拠（画像ありルートのみ）／AI 任せ
2. **線画の太さ統一**：太線（4〜6px相当・推奨）／中線（2〜3px）／細線（1px）／線なし
3. **影・ハイライト方針**：完全フラット（影なし）／単色影1段／セルシェード2段／グラデーション影
4. **共通背景処理**：完全透明（必須・LINE仕様）／キャラ周囲に微妙な発光は許容するか／白フチ（縁取り）の有無

→ ここで決まった画風が**全40枚で完全統一**されることを設計書に明記する。

## フェーズ5：画像生成AI 用プロンプト群の生成（最重要セクション）

設計書には以下を**全枚数分**機械的に展開する。

### 5-1. 共通プロンプト（全枚数共通の前置き）

#### ChatGPT (GPT-4o image / DALL-E) 用 — 自然言語

```
Create a LINE sticker illustration. Output specifications:
- Size: 370x320 pixels, PNG with fully transparent background
- Minimum 10px clear margin around the character on all sides; the character must NOT touch any edge
- Original character only — no copyrighted IP, no real persons, no brand logos, no LINE official marks
- Style: <フェーズ4で確定した画風記述（line weight, shading, palette）>
- Character: <フェーズ1で確定したキャラ仕様（種族・体型・色・特徴・パーツ）>
- Consistent character design across all stickers in the set
- No violence, no sexual content, no discrimination, no political/religious statements, no QR/barcodes, no third-party brand names
```

#### Stable Diffusion / NovelAI 用 — タグ列

```
Positive: original character, line sticker, transparent background, isolated on transparent, 370x320, 10px padding all sides, <キャラタグ>, <画風タグ>, simple flat colors, clean lineart, centered composition, full body or upper body, expressive pose, masterpiece, best quality
Negative: copyrighted character, real person, brand logo, watermark, signature, text artifact, blurry, low quality, jpeg artifacts, cropped, out of frame, extra limbs, bad anatomy, NSFW, gore, weapons, blood, political symbols, religious symbols, qr code, barcode, third-party logo, LINE official mark
```

### 5-2. 各スタンプ個別プロンプト（40枚分）

設計書テーブル形式で展開：

```markdown
### スタンプ #01 — <カテゴリ>：<台詞 or 無言>

**シーン記述**：<ポーズ・表情・小道具の具体>

**ChatGPT/DALL-E 用**：
```
[共通プロンプト] + The character is <ポーズ・表情の自然言語記述>. <台詞があれば>The text "<台詞>" appears in <配置> with <フォント方針> in <色>, clearly readable, no spelling errors.
```

**Stable Diffusion / NovelAI 用**：
Positive: [共通Positive] + <ポーズタグ>, <表情タグ>, <台詞があれば> text "<台詞>", <フォントタグ>
Negative: [共通Negative]
```

40枚分これを繰り返す（フェーズ2-Bで「全枚数自動生成」を選んだ場合は機械的に埋め、「主要のみ確認」を選んだ場合は確認した分は具体的に、残りはテンプレ枠で出して後で埋められるようにする）。

### 5-3. メイン画像・トークタブ画像 用プロンプト

| 画像 | サイズ | プロンプト |
|---|---|---|
| メイン画像 | 240×240 | フルカラー・キャラの最も特徴的なポーズ。スタンプセット全体の顔。 |
| トークタブ画像 | 96×74 | 単色シルエット推奨。小サイズでも識別できるシンプル構図。 |

それぞれに ChatGPT用・SD用の両方を生成する。

## フェーズ6：書き出し制作・申請チェックリスト（設計書末尾固定）

設計書末尾に以下のチェックリストを固定で書き込む。

### 画像生成後の処理

- [ ] 全画像の透過確認（Photoshop / Affinity Photo / Pixelmator で確認）
- [ ] 全画像のサイズ確認（370×320 内に収まり、10px以上の余白あり）
- [ ] 全画像のファイルサイズ確認（1MB 以内）
- [ ] 画風統一性の目視確認（40枚並べて違和感なし）
- [ ] テキストの誤字脱字確認
- [ ] メイン画像（240×240）作成
- [ ] トークタブ画像（96×74）作成

### LINE Creators Market 申請

- [ ] 販売情報入力（タイトル・説明文・タグ）
- [ ] スタンプ説明文に著作権侵害なきことを明記
- [ ] コピーライト表記（© <作者名> <年>）
- [ ] テストデバイスで実際にトーク表示プレビュー確認
- [ ] 価格設定（120円〜 / クリエイター取り分 50%）
- [ ] 申請送信 → 審査（通常 1〜7 営業日）

### 過去事故防止メモ

- 既存IPに似てしまった場合 → リジェクト確定。デザインから作り直し
- 余白不足（10px未満） → 自動リジェクト。生成時に必ずパディング指定
- 半透明ピクセル混入 → リジェクト要因。アンチエイリアスかけすぎに注意（生成プロンプトに `crisp edges, no semi-transparent pixels` を追加可）

## 量産モードのための追加ルール

- **連続生成要求があった場合**（「もう1セット作って」「次のスタンプ」等）、前回の設計書のパスを提示した上で、新しいスタンプ名を聞いてフェーズ0からやり直す。
- **同名スタンプの再生成要求**時、既存 `~/stamp-specs/<スタンプ名>.md` があれば上書き可否を確認する。
- **テンプレ再利用**：「前の○○と同じキャラで別シリーズ」と言われたら、既存設計書を Read してフェーズ1（キャラ設定）を流用し、フェーズ2以降のみ新規ヒアリングする。

## 呼び出し時の最初の発話

このスキルが起動したら、まず一文で「LINE スタンプ設計書を対話で作ります。まずスタンプ名と、キャラ画像の有無から確認します」と述べ、即 AskUserQuestion を呼ぶ。前置き長文は不要。
